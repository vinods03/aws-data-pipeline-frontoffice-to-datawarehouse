An SQS queue - orders-staging-queue - is created, with its access policy modified to be able to accept messages from the S3 bucket into which the transformed data is written into - vinod-streaming-project-staging-bucket. Replace aws-account-num with the actual aws account number.

{
  "Version": "2012-10-17",
  "Id": "123",
  "Statement": [
    {
      "Sid": "456",
      "Effect": "Allow",
      "Principal": {
        "Service": "s3.amazonaws.com"
      },
      "Action": "SQS:SendMessage",
      "Resource": "arn:aws:sqs:us-east-1:aws-account-num:orders-staging-queue",
      "Condition": {
        "StringEquals": {
          "aws:SourceAccount": "aws-account-num"
        },
        "ArnLike": {
          "aws:SourceArn": "arn:aws:s3:*:*:vinod-streaming-project-staging-bucket"
        }
      }
    }
  ]
}

In the S3 bucket -> properties, create an event notification for all object create events with destination as the above queue.
